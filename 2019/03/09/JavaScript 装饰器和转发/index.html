装饰器模式

装饰器是一个特殊的函数，它接受一个函数作为参数，并改变它的行为。

它非常重要和普遍的一种应用就是缓存函数返回结果，节省重复计算的时间。

call 和 apply 都允许显式地设置调用函数执行时的上下文（即 this 值），在作用上它们几乎相同。

不同的是 call 以参数列表的形式接收参数，而 apply 以类数组的形式接收参数。
